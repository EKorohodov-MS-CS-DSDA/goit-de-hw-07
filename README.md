# goit-de-hw-07
Згідно умов завдання, було виконано наступні кроки:

1. Було створено DAG з наступними задачами:
-  create_table - задача, що використовує SQLExecuteQueryOperator, та об'єднує в собі перевірку на існування та створення схеми та таблиці для збереження записів про медалі.
- pick_medal_task - задача виду BranchPythonOperator для вибору наступної задачі обираючи випадкове значення зі списку ['Gold', 'Silver', 'Bronze'].
- calc_Bronze, calc_Silver, calc_Gold - задачі виду SQLExecuteQueryOperator, що додають запис про один з видів медалей до таблиці.
-  generate_delay - задача виду PythonOperator для створення затримки відповідно до умови завдання.
-  check_for_correctness - завдання виду SqlSensor, що перевіряє, чи відбувся запис до таблиці протягом минулих 30 секунд. 

2. Налаштовано послідовність викликів задач.

3. Оскільки задача виконувалась локально, було розгорнуто відповідне середовичще
- встановлено необхідні пакети
- встановлено Airflow для standalone запусків
- запущено Docker контейнер olympic-mysql
- налаштовано з'єднання Airflow до бази у olympic-mysql контейнері
